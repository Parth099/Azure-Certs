# Lambda@Edge

- lightweight compared to actual `lambda`
- supports less features
- runs in AWS Public space as it has no VPC integration
- different limits compared to actual `lambda`

## Terms Used

- `viewer-request` - viewer requests content
- `origin-request` - Cloudfront reqeusts content from `origin`
- `origin-response` - reply of `origin-request`
- `viewer-response` - reply of `viwer-request`

## Lambda@Edge

The `Lambda@Edge` service can be run in each of four portions shown above to modify the request before it reaches the next stage.

### Uses Cases Samples

- viewer-request stage can be setup with a function to create A/B testing
- origin-request stage can be setup with a function to return different objects based on device type
